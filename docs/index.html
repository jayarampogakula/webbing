Webbing SaaS v0.8.0

Enterprise AI Website Builder SaaS

Welcome to Webbing—the production-ready, multi-tenant AI Website Builder.

🚀 Product Features

  • Multi-Tenant Workspace Isolation: Isolated database schemas, custom plan constraints, and active tenant metering.
  • AI Website Orchestrator: Generates multi-page websites (landing layout, features grids, SEO options) instantly using prompt directions.
  • Dynamic Custom Domains: Fast subdomain routing alongside on-demand automatic SSL cert configurations for client custom domains.
  • BullMQ Queued Worker: Background job execution consumer utilizing Redis storage to handle complex prompt workflows.
  • eCommerce Storefronts: Includes shopping carts, catalog items manager, product edit panels, and multi-image galleries.
Distribution Ready: This package contains configurations and license validators specifically tuned for CodeCanyon upload and instant host activation.

System Requirements

Ensure your server environment matches the hardware and database specifications detailed below.

💻 Hardware Requirements

  • Minimum Host: 1 vCPU, 2 GB RAM (for light testing workloads)
  • Recommended Host: 2 vCPUs, 4 GB RAM (with swap space activated if running postgres, redis, and next-js on the same node)
  • Disk Space: 10 GB+ SSD (dependent on database growth and self-hosted project packages)

🛠️ Software Requirements

  • OS: Ubuntu 20.04 LTS or newer (recommended), Debian, or WSL2 for Windows
  • Node.js: v20.x or newer (LTS version)
  • NPM: v10.x or newer
  • Database: PostgreSQL 14 or newer
  • Cache & Queue: Redis 6.2 or newer

🔑 External API Credentials

To use AI generation and online checkout, configure accounts with the following:

  • OpenAI / Gemini / Anthropic Keys: Powers copywriting and layout generation.
  • Stripe: Developer credentials for subscriptions and billing checkouts.
  • SMTP Mail Server: host port, user, and password to dispatch user invitations and password resets.

VPS Installation Guide

We provide two deployment options: Docker Compose (recommended) or manual PM2 Node processes.

🐋 Option 1: Docker Compose (Recommended)

Boot the web app, background queues, Redis cache, and Postgres DB inside detached containers.

1. Install Docker on your VPS

sudo apt update
sudo apt install -y docker.io docker-compose
sudo systemctl start docker
sudo systemctl enable docker

2. Setup Project Files

git clone https://github.com/jayarampogakula/webbing.git
cd webbing
cp .env.example .env
nano .env

3. Boot Detached Services

docker-compose up -d --build

🛠️ Option 2: Manual PM2 Deployment

Deploy using host-native Node.js services managed via PM2 process manager.

1. Setup Node 20 & PM2

curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt-get install -y nodejs postgresql redis-server
sudo npm install -y pm2 -g
sudo systemctl start postgresql redis-server

2. Install, Migrate and Seed

npm install --legacy-peer-deps
npm run db:generate
npx prisma db push --schema=packages/db/prisma/schema.prisma
node packages/db/prisma/seed.js
npm run build

3. Boot PM2 Instances

pm2 start "npm run start -w webbing-web" --name "webbing-web"
pm2 start "node packages/worker/dist/index.js" --name "webbing-worker"
pm2 save
pm2 startup

Setup Screen & SaaS Licensing

📍 Initial Configuration

Upon first launching Webbing on your custom domain, the platform detects that default user tables are unconfigured or using standard credentials. It automatically redirects you to:
http://yourdomain.com/setup

Fields Required:

  1. SaaS Name: Define your public application brand name.
  2. CodeCanyon Purchase Code: Enter the Envato Purchase Code received upon buying the product.
  3. Admin Details: Set custom email, full name, and secure passwords.

🔑 How Licensing Works

License key verification is handled inside apps/web/src/lib/licensing.ts.

Validation Methods:
  • CodeCanyon Purchase Code: Checked locally via regex formatting matching standard UUIDs: xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx.
  • Proprietary Activation Keys: Matches custom direct seller license keys: WEBBING-XXXX-XXXX-XXXX-XXXX.

🛠️ License Key Generation

If you wish to sell the SaaS directly to clients off-market, you can generate active license keys:

  1. Log in as Administrator and go to **Admin Console > SaaS Licensing**.
  2. Under **License Key Generator**, click **Generate Key**.
  3. A proprietary activation token (e.g. WEBBING-A4F2-8B1E-C6D9-F0A3) will be created. Copy and deliver this key to your client to input in their /setup wizard.

API Keys & LLM Configurations

Webbing supports a split keys mechanism to divide credit usages between administrators and standard hosts.

🤖 Admin Global Keys

Global keys are used to power AI generations for free-tier users or users without custom credentials. Define them in two ways:

1. Environmental Variables

Fill in key values inside the root .env file:

OPENAI_API_KEY="sk-proj-..."
GEMINI_API_KEY="AIzaSy..."
ANTHROPIC_API_KEY="sk-ant-..."

2. Admin Console UI

Go to **Admin Console > API Keys** and click **Add API Key**. Select `GLOBAL` scope. Keys configured in the database take precedence over .env variables.

Security Shield: Global administrator API keys are filtered out at the server level. They are never sent to frontend interfaces or standard client dashboard views.

👤 User Custom Keys & Bypass

Standard clients can go to their **Settings** panel and add their own private LLM provider keys (OpenAI or Gemini). When active:

  • Website generation requests bypass the administrator's global LLM accounts.
  • The client's **credit checks are bypassed**, allowing unlimited edits without drawing from their SaaS subscription package limits.
